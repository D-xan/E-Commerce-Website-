Project Overview & Objectives

This dashboard presents the comprehensive Quality Assurance strategy for the MockMart E-Commerce platform. The application is a custom-built testing ground designed to demonstrate enterprise-level automation engineering using Playwright and the Page Object Model (POM) pattern. Strategic Objectives

📏

End-to-End Coverage

Achieve complete execution of 120 test cases meticulously distributed across 8 core e-commerce services.

🏗️

Maintainability (POM)

Implement strict Page Object Model architecture to ensure test scripts are resilient to UI changes.

⚡

Speed & Reliability

Utilize Playwright's auto-waiting capabilities and parallel execution features for rapid, flake-free feedback.

📊

Data-Driven Testing

Separate dynamic test logic from static test data using structured, externalized JSON files.

🛡️

Defect Prevention

Proactively identify UI anomalies, logical business flaws, and vulnerabilities (like XSS) prior to simulated deployment.

Test Case Distribution Analytics

This section visualizes the mathematical balance of our testing scope. To prevent test bloat and ensure comprehensive coverage, exactly 15 test cases are allocated per microservice, categorized by testing scenario type: Positive (happy path), Negative (error handling), and Edge/Boundary cases.

Scenario Breakdown per Microservice

✅ Positive Scenarios

Verifying the system behaves correctly when provided with valid inputs and expected user flows. Ensures core business functionality.

⛔ Negative Scenarios

Testing system resilience against invalid inputs, unauthorized access, and incorrect workflows. Ensures proper error messaging.

⚠️ Edge/Boundary Cases

Pushing the system to its logical limits (e.g., extremely large inputs, zero values, timeout limits) to uncover hidden defects.

Framework Architecture & Execution

A clean, scalable architecture is the backbone of this QA project. We utilize a strict separation of concerns, keeping page locators separate from test logic, and test data separate from test scripts. This section details the file structure and execution commands.

🚀 Execution Commands

Run all tests (Headless):

npx playwright test

Run in UI Mode:

npx playwright test --ui

Run specific suite (e.g., Auth):

npx playwright test tests/auth.spec.js

View HTML Report:

npx playwright show-report
